# Session Index

All sessions archived here. One file per session, named `YYYY-MM-DD_S###_keywords.md`.

| Session | Date | Keywords | Summary |
|---------|------|----------|---------|
| S001 | [YYYY-MM-DD] | [keyword1-keyword2] | [one line summary] |

---

## File Format

Each session file uses YAML frontmatter:

```yaml
---
date: YYYY-MM-DD
session: S###
keywords: [keyword1, keyword2, keyword3]
components: [component1, component2]
summary: One sentence summary of what happened
status: archived
---
```

Followed by session notes, decisions, and context in Markdown.
